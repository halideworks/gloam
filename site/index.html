GLOAM
SDR-in-HDR Gamma Override

Fix the gray that Windows HDR casts over everything.

Gloam is a free, open-source Windows app that restores proper gamma to SDR content in HDR, with a color considered night mode.

A simple small tray app, designed by a colorist, that repairs the broken Windows HDR tone curve, per monitor, while you play, watch, or work.

01 / The problem

Windows lifts shadows and it's ugly.

When you enable HDR, Windows 11 decodes standard SDR window frames using a piecewise sRGB curve. Because almost all games, movies, and desktop assets are mastered for traditional gamma 2.2 screens, this curves mismatch dumps extra light directly into shadow regions.

In dark scenes, such as dark gameplay in Diablo IV, blacks turn into a flat gray haze, ruining ambient contrast in the areas where human vision is most sensitive:

+58.7% Extra light at 10% signal (deep shadows)
+52.5% Extra light at 20% signal (low mid-tones)
+34.3% Extra light at 30% signal (average shadows)

03 / What it does

One app. Three things Windows gets wrong.

Gloam not only fixes HDR/SDR gamma, but also replaces Night Light with a color-science-aware night mode and provides a full HDR capable calibration path when you are ready to measure.

HDR gamma fix

Per-monitor gamma 2.2, 2.4, or Windows default. Switchable per monitor with a hotkey. Self-healing ramp guard restores the correction if a game or driver resets it.

Night mode, done right

Temperature shifts are blended in linear light, where light physically mixes so night mode looks better. Sunset and sunrise are computed from the sun's position at your location, with programmable smooth fades and per-app exclusions.

Colorimeter calibration

Add a meter and Gloam installs a native Windows MHC2 profile, then verifies it by measuring through the applied correction. HDR verifications include a PQ tracking sweep in absolute nits.

Per-monitor dashboard

Real monitor names, HDR/SDR status, smart brightness levels, temp/tint and RGB adjustment, and live gamma mode readout for every connected display. Settings persist per monitor and restore on startup.

Native apply

LUTs write directly to the GPU gamma ramp through the Windows API. No helper process. No repeated rewrites when nothing has changed.

No telemetry, fully local

Your settings live in a local JSON file. The update checker is the only network call, and it only asks GitHub for the latest release tag.

04 / Built in the open MIT licensed

Free software, made to help.

Gloam exists because a lot of people bought beautiful HDR displays and then watched Windows wash them out. As a Windows user this bothers me but as a colorist this dismays me. The fix is straightforward enough to give away, and open enough that anyone can inspect it, improve it, or fork it.

The source is on GitHub. Builds are published through GitHub Releases. There is no paid tier, no subscription, no data collection, and no upsell. If you have a colorimeter, great. If you do not, the app still will make SDR content look pristine on your HDR display.

05 / Get it

Download Packages

One self-contained app, two ways to get it. Run the installer (recommended) or grab the portable zip, then enable "Start with Windows" from the tray menu if you want to correct your display on every boot.

Installer

Gloam-<version>-Setup.exe. The recommended way in.

  • Self-contained: no .NET runtime to install
  • Bundles ArgyllCMS, so colorimeter calibration works immediately
  • Per-user install, no administrator prompt
  • Updates itself in the background: download it once

Portable

Gloam-<version>-Portable.zip. No install, no traces.

  • Also self-contained: no .NET runtime to install
  • Extract it anywhere and run the executable
  • Does not auto-update: replace it with a newer zip to upgrade

06 / Questions

Before you ask

Do I need a colorimeter? +

No. The core gamma fix is pure math and needs no hardware. Night mode, per-monitor white point trims, and the dashboard all work without a meter. A colorimeter is only required for measurement-based calibration and verification.

Does it work in SDR, or only HDR? +

Both. HDR is where the gray haze is most obvious, so that is where Gloam started. The gamma fix operates inside the HDR signal path, and colorimeter corrections install as native Windows MHC2 profiles that the compositor applies in HDR and SDR. SDR displays get gamma control, night mode, and full calibration too.

How is this different from Windows Night Light? +

Three differences. First, the math: Gloam blends temperature in linear light, where light physically mixes, and its accurate CIE 1931 mode keeps the white point on the Planckian locus. Night Light is a simple compositor filter that tints without discernment.

Second, composition: Night Light fights other corrections for control of the pipeline. Gloam bakes the warm shift into the same high-precision LUTs as your gamma and calibration, so nothing bands or flickers.

Third, scheduling: sunset and sunrise are computed from the sun's actual position at your location, with configurable fades and levels (set different temps at different times), per-app exclusions, and per-monitor temperature offsets.

Which colorimeters work? +

Any meter supported by ArgyllCMS. The X-Rite i1 Display family and its Calibrite successors are the common path. Use a spectral correction matched to your panel; Gloam's in-app browser downloads .ccss and .ccmx files from the DisplayCAL community database. On QD-OLED and wide-gamut panels this is essential.

How is this different from DisplayCAL or Calman? +

For SDR desktop calibration, DisplayCAL is mature and excellent, and Calman is a professional standard for TV and mastering workflows. If those fit your use case, use them.

Gloam's focus is the HDR-native pipeline on Windows 11. Traditional tools apply corrections through the GPU gamma ramp and ICC profiles that the Advanced Color compositor largely ignores when HDR is on. Gloam builds a properly domain-wrapped MHC2 matrix plus PQ-domain tone LUTs, installs them through the association the HDR pipeline reads, and verifies by measuring through the applied profile. It's just a better option for most users using HDR without expensive professional software (though our math is built on the same principles).

Is it really free and open source? +

Yes. Gloam is MIT licensed and the source is on GitHub. ArgyllCMS, which Gloam invokes as a separate process for measurement, is AGPL v3. Gloam does not link against it.
