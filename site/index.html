Gloam

HDR-native display calibration for Windows, from the black floor to peak white.

Gloam measures your display with a colorimeter and installs the correction where Windows actually applies color: the HDR compositor itself, as a native MHC2 profile. No assumptions about how Windows maps SDR content, no profile that quietly stops working when HDR turns on. Measured, applied, and verified, with a scored report at the end.

Download for Windows Quick-start guide

Free and open source (MIT). Windows 10/11, HDR and SDR displays. Colorimeter optional for gamma correction, required for full calibration.

What it does

Windows 11 composites SDR content into HDR with a transfer function almost no content was mastered for, and most calibration tools were never built for the HDR pipeline at all. Gloam works inside that pipeline instead of around it.

Calibration

Colorimeter-driven, ArgyllCMS-based

Measure with an X-Rite i1 Display or any ArgyllCMS-supported meter. Gloam runs the whole flow hands-free: measure, build the correction, apply it, and re-measure through the applied profile to prove it worked.

HDR-native

MHC2/PQ pipeline, wire-exact

Corrections install as a native Windows MHC2 color profile applied by the compositor, persistently, in HDR and SDR. HDR runs measure true HDR-range patches through a DirectX FP16 surface at exact PQ wire positions, far above SDR white.

Tone

Per-monitor gamma with hotkeys

Gamma 2.2, 2.4 (BT.1886), or Windows default, set independently per monitor and switchable with global hotkeys. Fixes the washed-out shadows of Windows' SDR-in-HDR mapping. A ramp guard silently restores the correction if a game or driver resets it.

Comfort

Night mode with scheduling

Automatic warm-shift keyed to sunset and sunrise at your location, with smooth fades, per-app exclusions, and per-monitor temperature offsets. Baked into the correction pipeline, so it never fights Windows Night Light or your calibration.

Proof

Detailed verification

Native vs. calibrated dE2000 with histograms, worst and best patches, grayscale and primaries breakdowns, and corrected-response charts. HDR verifications add a PQ tracking sweep graded in absolute nits against ST.2084 and dE ITP (BT.2124).

Records

Report history and PDF export

Every calibration produces a scored report that is kept in history, so you can track a panel as it ages. Printer-friendly PDF export for sharing or archiving.

Scale

Multi-monitor dashboard

A grid view of every connected display with real EDID names, HDR/SDR status, current gamma mode, and quick access to settings and calibration. Settings persist per monitor and restore on startup.

Accuracy

Meter spectral corrections

An in-app browser for the DisplayCAL community corrections database (.ccss/.ccmx) matches your meter to your panel's emission spectra. Essential on QD-OLED and wide-gamut displays, where generic corrections misread the white point by several dE.

Questions

Which colorimeters work?

Any meter supported by ArgyllCMS, which Gloam uses for measurement. The X-Rite i1 Display family (i1 Display Pro, i1 Display Studio, and the Calibrite successors) is the well-trodden path. Spectrophotometers supported by Argyll work too.

Whatever the meter, use a spectral correction matched to your panel type. Gloam's built-in browser searches the DisplayCAL community corrections database and downloads .ccss/.ccmx files directly. On QD-OLED and wide-gamut panels this is not optional: generic corrections can misplace the white point by several dE.

Does it work in HDR? In SDR?

Both. HDR is the reason Gloam exists: the correction installs as a native MHC2 profile that the Windows compositor applies persistently, including with HDR enabled, and HDR calibrations measure true HDR-range patches at exact PQ wire positions instead of guessing how Windows maps SDR content onto the wire.

SDR displays are fully supported as well, including gamma control, night mode, and full colorimeter calibration.

Is it free? Open source?

Yes and yes. Gloam is MIT licensed and the source is on GitHub. ArgyllCMS, which Gloam invokes as a separate process for measurement, is AGPL v3; Gloam does not link against it.

Anything special about QD-OLED?

Two things. First, QD-OLED emission spectra are a pathological case for generic colorimeter corrections; use a QD-OLED .ccss from the in-app corrections browser or your readings will faithfully reproduce your meter's misplaced D65.

Second, if your panel already measures close to target natively (many QD-OLEDs do), prefer the white-point-only correction mode, which Gloam auto-suggests for OLED. Full gamut correction cannot improve what is already inside measurement noise, and verification will reliably come back worse if you try. Warm the panel up before measuring.

How is this different from DisplayCAL or Calman?

Honestly: for SDR desktop calibration, DisplayCAL is mature, excellent, and more configurable, and Calman is the professional standard for TV and mastering workflows. If that is your use case, use them.

Gloam's differentiator is the HDR-native pipeline on Windows 11. Traditional tools apply corrections through the GPU gamma ramp (VCGT) and ICC profiles that the Windows Advanced Color compositor largely ignores or resets when HDR is on. Gloam builds a properly domain-wrapped MHC2 matrix plus PQ-domain tone LUTs, installs them through the Advanced Color association the HDR pipeline actually reads, and verifies the result by measuring through the applied profile, including a PQ tracking sweep in absolute nits. It also stays resident to keep gamma preference, night mode, and the calibration composing cleanly instead of fighting each other.

Do I need a colorimeter at all?

No. The core feature that brought most users here, fixing Windows 11's washed-out SDR-in-HDR shadows with proper gamma 2.2/2.4 decoding, is pure math and needs no hardware. Night mode, per-monitor white point trims, and the dashboard all work without a meter. A colorimeter is only required for measurement-based calibration and verification.
